executeTransactionBlock

suspend fun executeTransactionBlock(txBytes: String, signatures: List<String>, options: TransactionBlockResponseOptions, requestType: ExecuteTransactionRequestType? = null): TransactionBlockResponse

Execute the transaction and wait for results if desired.

Request types: 1. WaitForEffectsCert: waits for TransactionEffectsCert and then return to client. This mode is a proxy for transaction finality. 2. WaitForLocalExecution: waits for TransactionEffectsCert and make sure the node executed the transaction locally before returning the client. The local execution makes sure this node is aware of this transaction when client fires subsequent queries. However, if the node fails to execute the transaction locally in a timely manner, a bool type in the response is set to false indicating the case. request_type is default to be WaitForEffectsCert unless options.show_events or options.show_effects is true

Return

TransactionBlockResponse The response from the transaction block execution.

Parameters

txBytes

BCS serialized transaction data bytes without its type tag, as base-64 encoded string.

signatures

A list of signatures (flag || signature || pubkey bytes, as base-64 encoded string). Signature is committed to the intent message of the transaction data, as base-64 encoded string.

options

for specifying the content to be returned.

requestType

The request type, derived from [TransactionBlockResponseOptions] if None.

Throws

SuiException

if the response is an error.
